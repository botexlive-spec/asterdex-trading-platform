4. BUSINESS LOGIC - ROI PLAN
================================================================================

TYPE: Individual earnings (not network-based)
CALCULATION: Fixed daily percentage on investment
FREQUENCY: Daily distribution at 00:00 UTC
CAP: 200% of initial investment (2x)

ROI MECHANICS:
1. User invests $X in a package
2. Daily ROI = $X × daily_roi_percentage ÷ 100
3. ROI distributed daily via cron job
4. Accumulates until total_roi_earned >= total_roi_limit (2 × investment)
5. Package status changes to 'completed' when limit reached
6. All ROI credited to users.wallet_balance

EXAMPLE:
Investment: $5,000 (VIP Package)
Daily ROI: 5% = $250/day
Duration: 40 days
Total Payout: $250 × 40 = $10,000 (200%)

Day-by-Day:
Day 1: wallet += $250, total_roi_earned = $250
Day 2: wallet += $250, total_roi_earned = $500
...
Day 40: wallet += $250, total_roi_earned = $10,000 → status = completed

6. BUSINESS LOGIC - BINARY PLAN
================================================================================

TYPE: Network-based, team volume matching
STRUCTURE: Binary tree (max 2 children per node)
CALCULATION: Percentage of weaker leg volume
POSITIONS: Left leg, Right leg

BINARY TREE STRUCTURE:
           User A (Root)
            /      \
        Left        Right
       User B      User C
       /    \      /    \
      D      E    F      G

VOLUME CALCULATION:
- Left Volume = SUM(all left leg investments, recursive)
- Right Volume = SUM(all right leg investments, recursive)
- Matching Bonus = MIN(left_volume, right_volume) × matching_bonus_%
- Carry Forward = |left_volume - right_volume| (rolls to next period)

EXAMPLE:
User A binary tree:
• Left leg total: $20,000
• Right leg total: $35,000
• Weaker leg: MIN($20,000, $35,000) = $20,000
• Matching bonus: $20,000 × 10% = $2,000
• Carry forward: $35,000 - $20,000 = $15,000

Next Period:
• Left leg grows by $5,000 → $25,000 total
• Right leg starts at $15,000 (carry forward)
• New matching: MIN($25,000, $15,000) = $15,000
• Bonus: $15,000 × 10% = $1,500
